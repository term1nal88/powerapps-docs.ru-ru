# <a name="whats-new-in-powerapps"></a>Новые возможности PowerApps
Сведения об известных ограничениях см. в статье [Типичные проблемы и решения](common-issues-and-resolutions.md).

**Примечание.** Номера версий последних выпусков PowerApps различаются в зависимости от платформы. Чтобы гарантированно использовать последний выпуск PowerApps для вашей платформы, установите его из Магазина Windows, App Store или Google Play.

## <a name="release-20700-20701"></a>Выпуски 2.0.700–2.0.701
1. После первого [сохранения приложения](save-publish-app.md) по умолчанию дополнительные изменения будут автоматически сохраняться каждые две минуты.
2. Легко [создавайте правила](working-with-rules.md) для условного форматирования без написания выражений. Просто задайте условие и пропишите результаты прямо на холсте PowerApps.
3. Легко настраивайте формы, коллекции и таблицы данных, используя область данных, которая раскрывается на полную высоту при добавлении элемента управления.
4. Получайте контекстные подсказки при создании пустого приложения, создании приложения на основе шаблона, источника данных или с помощью SharePoint.

## <a name="release-20600-20601"></a>Выпуск 2.0.600–2.0.601
1. **Экспериментальная функция. Добавлены динамические формы с элементом управления "Формы сущности", с помощью которых пользователи могут просматривать и редактировать реляционные данные из Common Data Service и переходить по ним.**
   
    Дополнительные сведения см. в статье об [использовании элемента управления "Формы сущности"](entity-form-control.md).
2. **Более удобное определение источников данных с помощью значка для каждого типа соединения.**
   
    Когда на панели источника данных PowerApps Studio вы добавляете соединение, для типов соединений (таки как SharePoint, OneDrive и YouTube) отображаются разные значки. Рядом с добавленным соединением отображается значок, напоминающий вам о том, какие источники данных вы добавили. Например, для соединения Common Data Service отображается значок этого типа соединения с его именем и именем сущности.
3. **Более удобное чтение длинных формул благодаря элементам синтаксиса с цветовым кодированием.**
   
    Когда вы вводите формулу в строку формулы, разные элементы синтаксиса отображаются разными цветами, чтобы вам было удобнее читать длинные формулы. Таблицу элементов с соответствующими цветами см. в статье о [начале работы с формулами](working-with-formulas.md#formula-syntax).
4. **Отправка push-уведомлений из приложения или последовательности в любые другие приложения с помощью соединения "Уведомления PowerApps".**
   
    Чтобы воспользоваться преимуществами полностью настраиваемого интерфейса, выберите пользователей, которые будут получать уведомления, а затем передайте сообщение и параметры в целевое приложение. Дополнительные сведения см. в статье о [добавлении push-уведомлений в приложение](https://powerapps.microsoft.com/blog/add-push-notification-to-you-app-and-boost-usage-and-retention/).
5. **Переименование или удаление среды в центре администрирования.**
   
    Дополнительные сведения см. в статье об [администрировании среды](environments-administration.md).
6. **Создавайте настраиваемый соединитель, указав URL-адрес для определения OpenAPI.**
   
    Дополнительные сведения см. в статье о [регистрации и использовании настраиваемых соединителей](register-custom-api.md).

## <a name="release-20590-20591"></a>Выпуск 2.0.590–2.0.591
1. **Отображение данных в коллекции, которая изменяет размер в соответствии с элементами различной высоты.**
   
    Добавьте коллекцию переменной высоты, если у вас, к примеру, есть текст (например, описание продукта) разной длины. При таком подходе описания в вертикальной коллекции можно показать полностью без лишних пробелов.
   
    Пошаговые инструкции по отображению данных в коллекции см. в статье [Show a list of items](add-gallery.md) (Отображение списка элементов). Справочные сведения см. в статье [Gallery control](controls/control-gallery.md) (Элемент управления "Коллекция").
2. **Определение того, начинается ли текстовая строка с указанных знаков.**
   
    Используйте функцию [StartsWith](functions/function-startswith.md), чтобы пользователи могли, например, ввести несколько букв в поле поиска для фильтрации списка элементов на основе текста в начале поля.
3. **Определение того, соответствуют ли текстовые строки шаблону, указанному с использованием регулярного выражения.**
   
    Используйте функцию [IsMatch](functions/function-ismatch.md) для возврата значения true или false в зависимости от того, соответствует ли значение поля в записи, например, имени, адресу электронной почты, номеру телефона или номеру социального страхования США.
4. **Настройка порядка активации элементов управления при нажатии клавиши табуляции.**
   
    По умолчанию координаты X и Y для каждого элемента управления определяют последовательность вкладок. Чтобы указать другой порядок, задайте для свойства **TabIndex** одного или нескольких элементов управления значение в виде целого числа больше нуля. Например, если элемент управления имеет значение 1, он активируется до элемента управления, для которого задано значение 2 или больше. Вы можете сделать приложение [доступнее](properties-accessiblity.md), добавив подсказки и настроив оптимальную последовательность вкладок на каждом экране.
5. **Настройка толщины полосы с ползунком.**
   
    Пользователи изменяют значение элемента управления [Ползунок](controls/control-slider.md), перетаскивая маркер вправо или влево вдоль полосы.
6. **На устройствах Android список приложений обновляется нажатием кнопки, а не путем проведения пальцем.**
   
    Дополнительные сведения см. в статье [Использование PowerApps на телефоне или планшете](run-app-client.md).
7. **Исправления ошибок и улучшения производительности**.

    Функция **[ForAll](functions/function-forall.md)** вычисляет формулы для всех записей таблицы. Формула может рассчитать значение и (или) выполнить одно или несколько действий, включая изменение данных или работу с подключением.

    Функция **[возведения в степень](functions/function-numericals.md)** возвращает число, возведенное в степень. Эта функция эквивалентна функции [^ operator](functions/operators.md).
10. **Улучшенная сортировка и фильтрация между сеансами и устройствами**.
    
     Если отсортировать или отфильтровать список приложений на мобильном устройстве, этот список сохранится даже при закрытии PowerApps. Если отсортировать список приложений по дате их открытия (начиная с последнего), порядок сортировки будет точным, даже если приложение открывается на другом устройстве.
11. **Улучшенный механизм предоставления отзывов**.
    
     Чтобы задать вопрос, теперь в качестве альтернативы использованию значка улыбки в правом верхнем углу просто создайте запрос с помощью элемента **Контакт** на [странице поддержки](https://powerapps.microsoft.com/support/).
