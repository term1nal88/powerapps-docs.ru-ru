# <a name="responding-to-data-subject-rights-dsr-requests-for-customer-data-in-common-data-service-for-apps"></a>Ответ на запросы по правам субъектов данных (DSR) для данных клиентов в Common Data Service для приложений

## <a name="introduction-to-dsr-requests"></a>Общие сведения о запросах DSR
В рамках наших партнерских обязательств по ознакомлению с Общим регламентом по защите данных (GDPR) в ЕС мы разработали эту документацию, чтобы помочь вам подготовиться. В этой документации не только описывается, что мы делаем для подготовки к внедрению GDPR, но также предоставляются примеры шагов, которые можно предпринять сегодня вместе с корпорацией Майкрософт, чтобы обеспечить соответствие GDPR при использовании PowerApps, Microsoft Flow и Common Data Service для приложений.

Общий регламент по защите данных (GDPR) ЕС предоставляет пользователям (называемым в регламенте субъектами данных) права по управлению персональными данными, которые были собраны работодателем или другой службой или организацией (называемыми контроллером данных или просто контроллером). В GDPR персональные данные имеют широкое определение как любые данные, относящиеся к идентифицированному или идентифицируемому физическому лицу. GDPR предоставляет субъектам данных определенные права на свои персональные данные. Эти права включают получение копий персональных данных, запрос на исправление этих данных, ограничение их обработки, удаление или получение данных в электронном формате для перемещения к другому контроллеру. Формальный запрос субъекта данных к контроллеру на выполнение действий со своими персональными данными называется запросом субъекта данных.

В руководстве рассказывается, как использовать продукты, службы и административные инструменты Майкрософт, чтобы помочь клиентам контроллера находить персональные данные и предпринимать с ними действия для ответа на запросы DSR. В частности, это включает руководство по поиску персональных данных, которые находятся в облаке Майкрософт, получению к ним доступа и выполнению в отношении них действий. Ниже приведен краткий обзор процедур, изложенных в этом руководстве:

1. **Обнаружение**. Используйте средства поиска и обнаружения, чтобы легко находить пользовательские данные, которые могут быть предметом запроса DSR. Собрав соответствующие документы, можно выполнить одно или несколько действий DSR, описанных в следующих шагах, чтобы ответить на запрос. Кроме того, можно определить, что запрос не соответствует рекомендациям вашей организации по ответу на запросы DSR.

2. **Доступ**. Получите персональную информацию, которая находится в облаке Майкрософт, и, если требуется, сделайте ее копию, которую можно предоставить субъекту данных.

3. **Исправление**. Внесите изменения или выполните другие запрошенные действия в отношении персональных данных, если это применимо.

4. **Ограничение**. Ограничьте обработку персональных данных, по возможности удалив лицензии для различных веб-служб или отключив соответствующие службы. Вы также можете удалить данные из облака Майкрософт и сохранить их локально или в другом расположении.

5. **Удаление**. Окончательно удалите персональные данные, которые хранились в облаке Майкрософт.

6. **Экспорт**. Предоставьте электронную копию (в формате, пригодном для машинного чтения) персональных данных субъекту данных.

В каждом разделе этого руководства описываются технические процедуры, которые организация-контроллер данных может выполнить в ответ на запросы DSR персональных данных в облаке Майкрософт.

## <a name="common-data-service-customer-data"></a>Данные клиента Common Data Service

> [!IMPORTANT]
> Применяется к Common Data Service для приложений и Common Data Service (предыдущая версия)

Существует два типа служб Common Data Service (CDS) — CDS для приложений и предыдущая версия CDS, в каждой из которых предусмотрены разные процессы работы с персональными данными.

Тип среды CDS можно определить, выполнив следующие действия с [сайта PowerApps](https://web.powerapps.com):

1.  Выберите **Среда** из раскрывающегося списка среды.
2.  Перейдите к пунктам **Данные** > **Сущности**.

    Ваша среда — CDS для приложений, если отображаются эти объекты:

    ![Список сущностей PowerApps](./media/common-data-service-gdpr-dsr-guide/powerapps-entities-list.png)

    Ваша среда — предыдущая версия CDS, если отображаются эти сущности:

    ![Список сущностей PowerApps прежних версий](./media/common-data-service-gdpr-dsr-guide/powerapps-legacy-entities-list.png)

Определив тип среды CDS, выполните инструкции в соответствующих разделах этого документа, чтобы определить персональные данные.

> [!NOTE]
> Возможно, некоторые среды будут развернуты в CDS для приложений, а другие — в CDS (предыдущая версия), поэтому вам понадобится повторить процессы, описанные ниже, для каждой среды в вашей организации.

## <a name="user-personal-data-in-common-data-service-for-apps"></a>Персональные данные пользователей в Common Data Service для приложений

### <a name="prerequisites"></a>Технические условия
Для доступа к CDS необходимо создать пользователя из Центра администрирования Office 365 и назначить соответствующую лицензию пользователя Common Data Service.  Кроме того, требуется роль безопасности, прежде чем пользователь сможет начать работу с Common Data Service.

Персональные данные пользователя хранятся в Центре администрирования Office 365 и в системной сущности пользователя CDS.  В Центре администрирования Office 365 хранятся и обслуживаются определенные персональные данные пользователя (например, имя пользователя, UserID, номер телефона, электронная почта и адрес). Они синхронизируются с системной сущностью пользователя CDS во всех средах.  Системный администратор может обновить эти персональные данные только в Центре администрирования Office 365, затем эти атрибуты автоматически синхронизируются с CDS для приложений. Системный администратор также может создать пользовательские атрибуты для сохранения дополнительных персональных данных пользователей в системной сущности пользователя CDS.  Системный администратор вручную обслуживает эти пользовательские атрибуты и управляет ими в CDS.

При удалении пользователя из Центра администрирования Office 365 запись пользователя не удаляется автоматически из системной сущности пользователя CDS, чтобы избежать прерывания работы бизнес-приложений, которые могут быть важны для работы в вашей организации.  Системному администратору CDS потребуется предпринять действия для поиска и удаления персональных данных пользователя из CDS с помощью приложения.

Только пользователи с ролью глобального администратора Office 365 и ролью системного администратора Office 365 могут обнаруживать, уточнять, экспортировать и удалять данные (описано ниже).

### <a name="discover"></a>Обнаружение

Системные администраторы могут создать несколько экземпляров CDS.  Эти экземпляры можно использовать для пробной версии, среды разработки или рабочей среды.   Каждый из этих экземпляров имеет копию системной сущности пользователя со всеми пользовательскими атрибутами, которые мог добавить системный администратор, а также персональные данные пользователей, синхронизированные из Центра администрирования Office 365.

Системный администратор может найти список всех экземпляров CDS, перейдя в Центр администрирования Dynamics 365 из центра администрирования PowerApps.

В [центре администрирования PowerApps](https://admin.powerapps.com/):

1.  Перейдите на вкладку "Среды".
2.  В списке сред выберите среду.
3.  Щелкните ссылку Центра администрирования Dynamics 365.

    ![Подробные сведения о среде PowerApps](./media/common-data-service-gdpr-dsr-guide/powerapps-environment-details.png)

    Появится список всех экземпляров.

    ![Средство выбора экземпляра PowerApps](./media/common-data-service-gdpr-dsr-guide/powerapps-instance-picker.png)

Персональные данные пользователей CDS можно найти в следующих ресурсах:

|Ресурсы, содержащие персональные данные | Цель | Доступ к веб-сайтам | Программный доступ
| --- | --- | --- | ---
| Запись сущности | Системная сущность пользователя | [Центр администрирования PowerApps](https://admin.powerapps.com) | [Через веб-API](https://docs.microsoft.com/en-us/dynamics365/customer-engagement/developer/webapi/update-delete-entities-using-web-api#basic-update)
| Журнал аудита | Чтобы позволить клиентам определять ресурсы, созданные, измененные, удаленные пользователями, или такие, к которым они обращались, на уровне сущности. | [Центр администрирования PowerApps](https://admin.powerapps.com) | [Через веб-API](https://docs.microsoft.com/en-us/dynamics365/customer-engagement/developer/webapi/update-delete-entities-using-web-api#basic-update)

#### <a name="user"></a>Пользователь
Персональные данные пользователя, обслуживаемые и управляемые в Центре администрирования Office 365, хранятся в Azure Active Directory.  Управление этими персональными данными осуществляется в Центре администрирования Office 365 и они автоматически синхронизируются для всех сред CDS.  Системный администратор не может обновить эти персональные данные непосредственно в CDS, пока активен пользователь. Их следует обновлять непосредственно в Центре администрирования Office 365.  Дополнительные персональные данные (например, пользовательские атрибуты) можно добавить непосредственно в CDS. Их должен обслуживать системный администратор, а также управлять ими вручную.

Системный администратор CDS может найти пользователя и персональные данные, связанные с ним, выполнив следующие действия:

В [центре администрирования PowerApps](https://admin.powerapps.com/):

1.  Перейдите на вкладку "Среды".
2.  В списке сред выберите среду.
3.  Щелкните ссылку Центра администрирования Dynamics 365.
4.  Щелкните имя среды.
5.  Нажмите кнопку **Открыть**.
6.  Перейдите к пунктам **Параметры** > **Безопасность**.
7.  Щелкните **Пользователи**.
8.  Введите имя пользователя в поле ввода **поиска**.
9.  Нажмите кнопку **Поиск**.
10. Дважды щелкните имя пользователя.

    ![Форма пользователя PowerApps](./media/common-data-service-gdpr-dsr-guide/powerapps-user-form.png)

#### <a name="audit-history"></a>Журнал аудита
Когда [отслеживание аудита будет включено](https://docs.microsoft.com/dynamics365/customer-engagement/admin/audit-data-user-activity) для сущности в Common Data Service, персональные данные пользователя будут регистрироваться в журнале аудита вместе с событиями, выполняемыми пользователем.

### <a name="rectify"></a>Исправление

Если субъект данных попросил исправить персональные данные, которые находятся в данных вашей организации, вам и вашей организации необходимо определить, нужно ли выполнять такой запрос.  Исправление данных может включать такие действия, как изменение, редактирование или удаление персональных данных из документа или другого типа элемента.

Вы можете использовать Azure Active Directory для управления удостоверениями (персональные данные) пользователя в Common Data Service для приложений. Корпоративные клиенты могут управлять запросами на исправление DSR, включая ограниченные возможности редактирования в зависимости от характера данной службы Майкрософт.  В качестве обработчика данных Майкрософт не предлагает возможности исправлять системные журналы, так как это отражает фактические действия и представляет собой журнал событий в службах Майкрософт. Подробные сведения см. в статье о [запросах субъектов данных (сигналы DSR) в рамках GDPR](https://servicetrust.microsoft.com/ViewPage/GDPRDSR).

После удаления записи пользователя из Azure Active Directory системные администраторы могут удалить любые оставшиеся персональные данные пользователей (например, пользовательские атрибуты, которые могли быть добавлены) из всех экземпляров.

7.  Нажмите кнопку "Экспорт в Excel".

### <a name="delete"></a>Удалить

#### <a name="user"></a>Пользователь
Если пользователь удален из Центра администрирования Office 365, он имеет состояние "Отключен" в CDS. Однако персональные данные пользователя не удаляются автоматически, чтобы избежать прерывания работы бизнес-приложений, которые могут быть важны для работы в вашей организации.
Системному администратору требуется вручную удалить персональные данные отключенного пользователя из CDS.

#### <a name="remove-user-personal-data-via-user-form"></a>Удаление персональных данных пользователя через форму пользователя
При удалении записи пользователя из Azure Active Directory в форме пользователя отображается следующее сообщение.
Office 365 больше не управляет сведениями этого пользователя. Вы можете обновить эту запись для ответа на запросы DSR, удалив или заменив все персональные данные, связанные с этим пользователем.

В [центре администрирования PowerApps](https://admin.powerapps.com/):
1.  Перейдите на вкладку "Среды".
2.  В списке сред выберите среду.
3.  Щелкните ссылку Центра администрирования Dynamics 365.
4.  Щелкните имя среды.
5.  Нажмите кнопку **Открыть**.
6.  Щелкните **Параметры** > **Безопасность** > **Пользователи**.
7.  Выберите представление **Отключенные пользователи**.
8.  Введите имя пользователя в поле **Поиск записей** и нажмите кнопку **Поиск**.
9.  Дважды щелкните имя пользователя в результатах поиска.
10. Удалить все персональные данные и щелкните **Сохранить**.

#### <a name="remove-user-personal-data-via-excel-importexport"></a>Удаление персональных данных пользователя с помощью импорта и экспорта Excel
1.  Щелкните **Параметры** > **Безопасность** > **Пользователи**.
2.  Выберите представление **Отключенные пользователи**.
3.  Создайте шаблон Excel со всеми столбцами персональных данных пользователя, которые требуется обновить.
4.  Щелкните **Загрузить файл**
5.  Откройте скачанный Excel-файл, внесите изменения и сохраните файл.
6.  Вернитесь в представления отключенных пользователей и щелкните "Импорт данных".
7.  Выберите ваш обновленный Excel-файл в диалоговом окне "Отправить файл данных".
8.  Внесите требуемые изменения в окне сопоставления полей.
9.  Нажмите кнопку "Далее" и "Отправить".

Ознакомьтесь с [дополнительными сведениями об использовании шаблонов Excel](https://docs.microsoft.com/en-us/dynamics365/customer-engagement/admin/analyze-your-data-with-excel-templates).


#### <a name="remove-audit-history-via-the-audit-summary-view-form"></a>Удаление журнала аудита с использованием формы "Сводное представление аудита"

В [центре администрирования PowerApps](https://admin.powerapps.com/):
1.  Перейдите на вкладку "Среды".
2.  В списке сред выберите среду.
3.  Щелкните ссылку Центра администрирования Dynamics 365.
4.  Щелкните имя среды.
5.  Нажмите кнопку "Открыть".
6.  Щелкните "Параметры > Аудит".
7.  Выберите "Сводное представление аудита".
8.  Найдите журнал изменений пользователя.
9.  Установите флажок в соответствующих строках.
10. Щелкните значок удаления журнала изменений.

## <a name="personal-data-stored-in-common-data-service-for-apps-databases"></a>Персональные данные, хранимые в базах данных Common Data Service для приложений

### <a name="prerequisites"></a>Технические условия
Персональные данные отдельных пользователей (например, собственные клиенты) могут храниться в содержимом сущностей CDS.  

При подаче запроса DSR в вашу организацию системному администратору CDS необходимо найти все сущности, в которых может быть указан запросивший в приложениях.  Администратор CDS отвечает за инвентаризацию хранения персональных данных в различных используемых сущностях, чтобы вы могли отвечать на запросы DSR от отдельных лиц.

Персональные данные в пределах содержимого затем можно экспортировать, исправить или удалить в сущности с помощью функций в продукте.  

### <a name="discover"></a>Обнаружение
Получив запрос DSR от отдельного лица, администратору CDS понадобится определить среды или экземпляры CDS, которые содержат персональные данные этого лица.  Следует разработать политики и процедуры инвентаризации сред, экземпляров и сущностей, в которых вы решили хранить персональные данные лиц, для идентификации персональных данных в пределах содержимого.

Используя набор сред, экземпляров, сущностей и полей, где хранятся персональные данные, можно настроить средство поиска CDS для обнаружения персональных данных.  Администратор CDS может настроить поисковые сущности и поля. Дополнительные сведения см. в статье о [настройке поиска с сортировкой по релевантности](https://go.microsoft.com/fwlink/?linkid=872506).
После этого администратор CDS может получить доступ к среде CDS и выполнить поиск.

1.  Щелкните значок **Поиск**.
2.  Выберите **Поиск с сортировкой по релевантности**.

    ![Меню поиска с сортировкой по релевантности PowerApps](./media/common-data-service-gdpr-dsr-guide/powerapps-relevance-search-menu.png)

3.  В поле поиска введите персональные данные лица.
4.  Нажмите кнопку поиска.

    ![Результаты поиска с сортировкой по релевантности PowerApps](./media/common-data-service-gdpr-dsr-guide/powerapps-relevance-search-results.png)

Персональные данные в содержимом обычно хранятся в ключевых сущностях, например учетная запись, контакт, интерес, возможная сделка и т. д., но вы отвечаете за поддержку мест хранения персональных данных пользователей.

### <a name="rectify"></a>Исправление
Системный администратор CDS может обновить персональные данные пользователя, используя список результатов поиска с сортировкой по релевантности.  Однако персональные данные этого лица также могут храниться в других пользовательских объектах.  Системный администратор CDS отвечает за учет этих пользовательских сущностей и соответствующее обновление персональных данных лица.

В результатах поиска с сортировкой по релевантности:

1.  Щелкните элемент, в котором найдены персональные данные лица.
2.  При необходимости обновите персональные данные лица.
3.  Щелкните "Сохранить".

    ![Сведения об учетной записи PowerApps](./media/common-data-service-gdpr-dsr-guide/powerapps-account-details.png)

### <a name="export"></a>Экспортировать
Вы можете сделать снимок экрана с данными и предоставить к нему общий доступ для запросившей стороны DSR, сделав следующее.

В [центре администрирования PowerApps](https://admin.powerapps.com/):
1.  Перейдите на вкладку "Среды".
2.  В списке сред выберите среду.
3.  Щелкните ссылку Центра администрирования Dynamics 365.
4.  Щелкните имя среды.
5.  Щелкните значок **Поиск**.
6.  Выберите "Поиск с сортировкой по релевантности".
7.  В поле поиска введите персональные данные лица.
8.  Нажмите кнопку поиска.
9.  Найдите элемент и дважды щелкните его.
10. Нажмите клавишу <alt> <PrtScn> для создания снимка экрана.
11. Сохраните снимок экрана в файл.
12. Затем можно отправить файл инициатору запроса DSR.

### <a name="delete"></a>Удалить

Администратор CDS может удалить персональные данные пользователя из записей, где они хранятся.  Он может либо (1) удалить запись, где хранятся персональные данные, либо (2) удалить содержимое персональных данных из записи.  

> [!NOTE]
> Администратор CDS может настроить среду, чтобы предотвратить удаление записи из сущности. При этом необходимо удалить содержимое персональных данных из записи, а не саму запись.

В результатах поиска с сортировкой по релевантности:
1.  Щелкните элемент, в котором найдены персональные данные лица.
2.  Щелкните значок удаления на ленте (Примечание. Он будет отключен, если невозможно удалить запись.).

    ![Удаление учетной записи PowerApps](./media/common-data-service-gdpr-dsr-guide/powerapps-account-delete.png)

## <a name="personal-data-stored-in-common-data-service-previous-version-databases"></a>Персональные данные, хранящиеся в базах данных Common Data Service (предыдущая версия)

### <a name="prerequisites"></a>Технические условия

Персональные данные отдельных пользователей (например, собственные клиенты или пользователи) могут храниться в содержимом сущностей CDS.  

При подаче запроса DSR в вашу организацию системному администратору CDS необходимо найти все сущности, в которых может быть указан запросивший в приложениях.  Администратор CDS отвечает за инвентаризацию хранения персональных данных в различных используемых сущностях, чтобы вы могли отвечать на запросы DSR от отдельных лиц.

Персональные данные в пределах содержимого затем можно экспортировать, исправить или удалить в сущности с помощью функций в продукте.  

### <a name="discover"></a>Обнаружение
Получив запрос DSR от отдельного лица, администратору CDS понадобится определить среды или экземпляры CDS, которые содержат персональные данные этого лица.  Следует разработать политики и процедуры инвентаризации сред, экземпляров и сущностей, в которых вы решили хранить персональные данные лиц, для идентификации персональных данных в пределах содержимого.

#### <a name="entity-records"></a>Записи сущности
Персональные данные лица могут храниться в любой бизнес-сущности.
Эта версия Common Data Service содержит собственную схему и инфраструктуру базы данных.  Она имеет собственные сущности, управление которыми осуществляется на [сайте PowerApps](http://web.powerapps.com/).

Чтобы просмотреть список сущностей:

1.  Выберите среду.

    ![Список сред PowerApps прежних версий](./media/common-data-service-gdpr-dsr-guide/powerapps-legacy-entities.png)

2.  Перейдите к вкладке **Данные** > **Сущности**.
3.  Выберите сущность, например "Учетная запись".

    ![Список со сведениями о сущности PowerApps прежних версий](./media/common-data-service-gdpr-dsr-guide/powerapps-legacy-entities-details-list.png)

4.  Щелкните сущность.
5.  Щелкните вкладку **Данные**. Появится список записей для сущности.

    ![Данные учетной записи PowerApps прежних версий](./media/common-data-service-gdpr-dsr-guide/powerapps-legacy-account-data.png)

6.  Щелкните значок **Экспорт данных**.
7.  Откройте лист Excel после завершения экспорта.
8.  Щелкните поле "Включить правку".
9.  Щелкните значок поиска.
10. Введите персональные данные, которые нужно найти.
Персональные данные в содержимом обычно хранятся в ключевых сущностях, например учетная запись, контакт, интерес, возможная сделка, рабочая роль и т. д., но вы отвечаете за поддержку мест хранения персональных данных пользователей.
11. Используя инвентарный список сущностей, где хранятся персональные данные, **повторите описанные выше шаги для каждой бизнес-сущности, чтобы обнаружить персональные данные лица**.

### <a name="rectify"></a>Исправление
Если субъект данных попросил исправить персональные данные, которые находятся в данных вашей организации, вам и вашей организации необходимо определить, нужно ли выполнять такой запрос.  Исправление данных может включать такие действия, как изменение, редактирование или удаление персональных данных из документа или другого типа элемента.

Вы можете использовать Azure Active Directory для управления удостоверениями (персональные данные) пользователя в Common Data Service для приложений. Корпоративные клиенты могут управлять запросами на исправление DSR, включая ограниченные возможности редактирования в зависимости от характера данной службы Майкрософт.  В качестве обработчика данных корпорация Майкрософт не предоставляет возможность исправлять системные журналы, так как они отражают фактические действия и представляют собой исторические записи событий в службах Майкрософт.  

Подробные сведения см. в статье о [запросах субъектов данных (сигналы DSR) в рамках GDPR](https://servicetrust.microsoft.com/ViewPage/GDPRDSR).

Чтобы уточнить персональные данные в среде CDS, можно экспортировать данные сущности на лист Excel, обновить его и импортировать изменения в базу данных.
Администратор CDS отвечает за определение всех сущностей, в которых хранятся персональные данные лица, и повторное выполнение шагов ниже для каждой из этих сущностей.

С [сайта PowerApps](http://web.powerapps.com/):

1.  Щелкните **Данные** > **Сущности**.
2.  Щелкните сущность, например "Учетная запись".
3.  Щелкните параметр **Данные**.
4.  Щелкните значок **Экспорт данных**.
5.  Щелкните значок **Открыть в Excel** (после завершения экспорта).
6.  **Включите редактирование** на листе Excel и измените персональные данные.
7.  **Сохраните** обновленный лист (используйте функцию **Сохранить как**, чтобы знать, где находится файл).

    ![Данные учетной записи PowerApps прежних версий](./media/common-data-service-gdpr-dsr-guide/powerapps-legacy-account-data.png)

8.  Внесение необходимых изменений в персональные данные
9.  Сохранение изменений
10. На форме "Данные > Сущности > Учетная запись" щелкните значок "Импорт данных".
11. Щелкните поле поиска.
12. Выберите файл с обновлениями.
13. Щелкните поле "Открыть".
14. Нажмите кнопку "Импорт".

### <a name="export"></a>Экспортировать

Вы можете просматривать персональные данные из каждой сущности и экспортировать их на лист Excel.

С [сайта PowerApps](http://web.powerapps.com/):

1.  Перейдите к пунктам **Данные** > **Сущности**.
2.  Выберите **сущность**, данные которой требуется просмотреть и экспортировать.
3.  Щелкните параметр **Данные**.
4.  Щелкните значок **Экспорт данных**. Эта операция экспорта выполняется в фоновом режиме. Вы получите уведомление о ее завершении.
5. Для просмотра экспортированных данных щелкните значок "Открыть в Excel".

### <a name="delete"></a>Удалить
Персональные данные, хранимые в сущностях, можно удалить с помощью функции экспорта и импорта данных.

Администратор CDS отвечает за определение всех сущностей, содержащих персональные данные лица, и повторное выполнение следующих шагов для каждой из этих сущностей.

С [сайта PowerApps](http://web.powerapps.com/):

1.  Щелкните **Данные** > **Сущности**.
2.  Прокрутите вниз список **сущностей** и найдите сущность, из которой необходимо удалить персональные данные.
3.  Щелкните сущность.
4.  Щелкните параметр **Данные**.
5.  Щелкните значок **Экспорт данных**.
6.  Щелкните значок **Открыть в Excel** (после завершения экспорта).
7.  **Включите редактирование** на листе Excel.
8.  **Сохраните** обновленный лист (используйте функцию "Сохранить как", чтобы знать, где находится файл).
9.  Удалите строки записей персональных данных, которые требуется удалить.
10. Сохранение изменений
11. В форме **Сущности** щелкните значок **Импорт данных**.
12. Щелкните поле **поиска**.
13. Выберите файл с обновлениями.
14. Щелкните поле **Открыть**.
15. Нажмите кнопку "Импорт".
