# <a name="model-driven-app-common-field-properties"></a>Свойства поля "Общие" приложения на основе модели

 В полях формы отображаются элементы управления, с помощью которых пользователи просматривают или изменяют данные в записи сущности. Поля можно отформатировать, так чтобы они занимали до четырех столбцов в разделе.  

В обозревателе решений можно получить доступ к **Свойства поля "Общие"**. В разделе **Компоненты** разверните **Сущности**, а затем разверните нужную сущность и щелкните **Формы**. В списке форм откройте форму типа **Главная**. Чтобы просмотреть свойства поля "Общие", дважды щелкните одно из полей.

![Свойства общего поля](media/common-field-properties.png)
  
В следующей таблице представлены свойства всех полей. Некоторые типы полей обладают специальными свойствами. Они описаны в статье [Общие сведения о свойствах специальных полей приложений на основе моделей](special-field-properties-legacy.md).  
  
|Вкладка|Свойство|Описание|  
|---------|--------------|-----------------|  
|**Отображение**|**Метка**|**Требуется**. По умолчанию метка будет соответствовать отображаемому имени поля. Чтобы переопределить имя формы, здесь необходимо указать другую метку.|  
||**Метка для отображения в форме**|Существует вариант, при котором метка не отображается.|  
||**Поведение поля**|Укажите уровень поведения для поля с помощью флажков.|  
||**Выполняется блокировка**|Позволяет предотвратить случайное удаление поля с формы. Это позволит не выполнять настройки, которые были применены к полю, например удаление обработчика событий при удалении поля. Чтобы удалить это поле, настройщику сначала потребуется очистить этот параметр.|  
||**Видимость**|Отображение поля является необязательным. Включение отображения может задаваться с помощью скриптов. Дополнительные сведения см. в статье [Отображение или скрытие элементов формы для приложения на основе модели](visibility-options-legacy.md)|  
||**Доступность**|Выберите, будет ли вкладка доступна на телефоне.|
|**Форматирование**|**Выберите число столбцов, которое занимает элемент управления**|Когда раздел, содержащий кнопку поля, имеет более одного столбца, можно указать, чтобы поле занимало несколько столбцов, вплоть до их числа, доступного в разделе.|  
|**Сведения**|**Отображаемое имя**, **Имя** и **Описание**|Поля со свойством "только для чтения" предназначены для рекомендаций. Чтобы получить удобный доступ к определению поля (если его требуется изменить), нажмите кнопку **Изменить**.<br /><br /> Каждый экземпляр поля в форме обладает свойством имени, что позволяет ссылаться на них в сценариях формы. А данное имя управляется приложением. Имя первого экземпляра поля соответствует имени, указному при создании. Дополнительные сведения см. в разделе [How to create and edit fields](../common-data-service/create-edit-fields.md) (Создание и изменение панелей мониторинга).<br /><br /> За каждый дополнительный раз, когда поле включено в форму, к имени будет добавлено число (начиная с 1 до конца). Поэтому если имя поля "new_cost", то именем первого экземпляра станет "new_cost", второго — "new_cost1" и т. д.<br /><br />**Примечание**. Значение поля **Описание** содержит текст подсказки, который появляется при наведении.|  
|**События**|**Библиотеки форм**|Укажите веб-ресурсы JavaScript, которые будут использоваться в поле обработчика события `OnChange`.<br /><br />|  
||**Обработчики событий**|Настройте те функции из библиотеки форм, которые должны вызываться для поля события `OnChange`. Дополнительные сведения см. в статье [Настройка обработчиков событий форм приложений на основе моделей](configure-event-handlers-legacy.md)|  
|**Бизнес-правила**|**Бизнес-правила**|Просмотр и управление любыми бизнес-правилами, которые ссылаются на это поле. Дополнительные сведения см. в статье [Руководство по созданию бизнес-правил и рекомендаций для применения логики в форме приложения на основе моделей](create-business-rules-recommendations-apply-logic-form.md).|  
|**Элементы управления**|**Элементы управления**|Добавьте элементы управления и укажите их доступность в Интернете, телефоне и планшете.|
