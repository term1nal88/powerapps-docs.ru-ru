# <a name="overview-of-model-driven-app-special-field-properties"></a>Общие сведения о свойствах специальных полей приложений на основе моделей

 Все поля имеют свойства, перечисленные в разделе [Общие свойства полей](common-field-properties-legacy.md), но некоторые поля имеют дополнительные свойства; например, это поле объема обслуживания, которое можно открыть из основной формы для сущности "Обращение".

## <a name="lookup-field-properties"></a>Свойства полей поиска  
  
> [!NOTE]
>  В следующей таблице описаны параметры, доступные только для полей поиска одной сущности.  
  
|Раздел|Свойство|Описание|  
|-------------|--------------|-----------------|  
|**Фильтрация связанных записей**|**Показывать только записи, где**|Если эта функция включена, к записям, отображаемым при поиске, будет применена дополнительная фильтрация. Это повышает релевантность поиска при задании значения поиска.<br /><br /> По умолчанию эта функция отключена.<br /><br /> Сочетания связей, возможные при фильтрации связанных записей, перечислены в таблице, следующей за этой*.<br /><br /> Первый список заполнен потенциальными связями, которые можно использовать для фильтрации такого поиска. Выберите одну из них.<br /><br /> Тогда второй список заполняется всеми связями между соответствующей сущностью (выбранной в первом списке) и целевой сущностью. Выберите одну из них.<br /><br /> Установите флажок **Разрешить пользователям отключать фильтр**, чтобы разрешить пользователям отключать здесь фильтр.<br /><br /> Когда пользователи выбирают параметр **Найти еще записи** при установке значения для поиска, они увидят это диалоговое окно.<br /><br /> ![поиск-других-записей](media/crm-ua-v-8-1-look-up-more-records.png) <br /><br /> Если вы выбрали параметр **Разрешить пользователям отключать фильтр** при настройке поля поиска, пользователи будут видеть флажок, позволяющий отключить фильтр.  Это позволяет им расширить видимый диапазон записей. Если вы хотите сделать так, чтобы пользователи видели только ограниченный набор записей, определенных фильтром, снимите флажок **Разрешить пользователям отключать фильтр**.|  
|**Дополнительные свойства**|**Отобразить поле поиска в диалоговом окне поиска**|Вы можете отключить отображение поля поиска в диалоговом окне поиска.|  
||**Представление по умолчанию**|Это представление используется для фильтрации результатов встроенного поиска и настройки представления по умолчанию, показываемого в диалоговом окне поиска, когда пользователь выбирает параметр **Найти еще записи**.<br /><br /> Представление по умолчанию также контролирует, какие поля должны быть включены во встроенный поиск.<br /><br /> При поиске, где разрешается выбрать только один тип сущности, отображаемые при встроенном поиске поля — это первые два поля, включенные в представление по умолчанию. В этом примере **Основной телефон** и **Адрес электронной почты** — первые два столбца в представлении по умолчанию, настроенные для поиска учетных записей.<br /><br /> При системном поиске, где разрешается выбрать несколько типов сущностей, отображаются первые два поля в представлении поиска сущностей.|

### <a name="two-option-field-properties"></a>Свойства полей с двумя вариантами  
 На вкладке форматирования для полей с двумя вариантами доступны следующие варианты форматирования:  
  
- **Два переключателя**: два помеченных элемента управления с метками. Можно выбрать только один.  
  
- **Флажок**: один флажок, позволяющий задать значение true или false.  
  
- **Список**: раскрывающийся список, содержащий оба значения.  
  
<a name="BKMK_MultipleLinesOfTextProperties"></a>   

### <a name="multiple-lines-of-text-field-properties"></a>Свойства многострочных текстовых полей  
 Многострочные и однострочные текстовые поля, использующие формат `Text Area`, имеют свойство **Макет строк**. С помощью этого свойства можно указать значение **Число строк** или выбрать параметр **Автоматически развертывать для заполнения доступного пространства**.
