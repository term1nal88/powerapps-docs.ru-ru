# <a name="import-update-and-export-solutions"></a>Импорт, обновление и экспорт решений 

 Решения можно импортировать вручную, выполнив следующие действия. Импортируйте только решения, полученные из надежного источника. Настройки могут содержать код, который может отправлять данные внешним источникам. Решение по умолчанию можно импортировать только в организацию, из которой это решение было экспортировано, но не в другую организацию.  
  
1. Выберите раздел **[Параметры](../model-driven-apps/advanced-navigation.md#settings)** > **Решения**.  
  
2.  В меню списка решений выберите **Импорт**.  
  
3.  В диалоговом окне **Импорт решения**, **Выбрать пакет решения** перейдите к сжатому файлу (ZIP или CAB), содержащему решение, которое нужно импортировать. 
  
4.  Выберите **Далее**.  
  
5.  Прежде чем выбрать **Импорт**, можно просмотреть сведения о решении.  
  
6.  Может понадобиться подождать несколько секунд, пока завершится импорт решения. После успешного импорта можно просмотреть результаты и выбрать **Закрыть**.  
  
 Если импортированы изменения, требующие публикации, необходимо опубликовать настройки, прежде чем они станут доступны. 
  
 Если импорт был неудачен, появится отчет, содержащий все зарегистрированные ошибки и предупреждения. Для просмотра сведений о причине сбоя импорта можно выбрать **Загрузить файл журнала**. Наиболее распространенной причиной сбоя импорта решения является то, что это решение не содержало ряд обязательных компонентов решения.  
  
 Загруженный XML-файл журнала можно открыть с помощью Office Excel, чтобы просмотреть его содержимое.  
  
> [!NOTE]
>  Активный набор правил маршрутизации редактировать нельзя. Таким образом, при импорте решения, содержащего активный набор правил маршрутизации, в организацию, где уже существует правило с таким же идентификатором, импорт решения завершится ошибкой. Дополнительные сведения: [Создание правил для автоматической маршрутизации заданий](https://docs.microsoft.com/dynamics365/customer-engagement/customer-service/create-rules-automatically-route-cases)

## <a name="update-solutions"></a>Обновление решений  
 Бывают случаи, когда может потребоваться установить обновление существующего управляемого решения. Процедура аналогична установке нового управляемого решения, за исключением получения некоторых отличающихся параметров. При обновлении решения, полученного от другого лица, следует получить указания издателя решения о том, какие параметры выбрать.  
  
1. Выберите раздел **[Параметры](../model-driven-apps/advanced-navigation.md#settings)** > **Решения**.   
  
2.  В меню списка решений выберите **Импорт**.  
  
3.  В диалоговом окне **Импорт решения**, **Выбрать пакет решения** перейдите к сжатому файлу (ZIP или CAB), содержащему решение, которое нужно обновить.  
4.  Выберите **Далее**.  
  
5.  Перед нажатием кнопки **Далее** можно просмотреть информацию о решении. Эта страница будет содержать желтую полосу с надписью **Этот пакет решения содержит обновление для уже установленного решения**.  
  
6.  Выберите один из следующих вариантов:  
  
    - **Сохранить настройки (рекомендуется)**  
  
         При выборе этого варианта позволит сохранить все неуправляемые настройки, выполненные для компонентов, но при этом ряд обновлений, включенных в это решение, не вступят в силу.  
  
    - **Заменить настройки**  
  
         При выборе этого параметра перезаписываются все неуправляемые настройки, выполненные ранее для компонентов, входящих в это решение. Все обновления, включенные в это решение, вступят в силу.  
  
     Выберите нужный вариант и нажмите кнопку **Далее**.  
  
7.  Может понадобиться подождать несколько секунд, пока завершится импорт решения. После успешного импорта можно просмотреть результаты и выбрать **Закрыть**.  
  
 Если импортированы изменения, требующие публикации, необходимо опубликовать настройки, прежде чем они станут доступны. 
  
 Издатели решений могут предложить экспортировать существующие неуправляемые настройки, обновить созданные ими управляемые решения, используя вариант с заменой настроек и импортировать неуправляемые настройки обратно. Это поможет гарантировать применение ожидаемых изменений при одновременном сохранении настроек.

## <a name="export-solutions"></a>Экспорт решений  
 Рекомендуется периодически экспортировать неуправляемые настройки, чтобы иметь резервную копию на всякий случай. Экспорт управляемых решений невозможен.  
  
1. Выберите раздел **[Параметры](../model-driven-apps/advanced-navigation.md#settings)** > **Решения**.   
  
2.  Выберите в списке решение, которое нужно экспортировать, и нажмите кнопку **Экспорт**.  
  
3.  На шаге **Публиковать настройки** появится напоминание о том, что экспортируются только опубликованные настройки и перед нажатием кнопки **Далее** есть возможность **Публиковать все настройки**.  
  
4.  Если в решении отсутствуют любые обязательные компоненты, появится шаг **Отсутствуют обязательные компоненты**. Этим предостережением можно пренебречь, только если планируется импортировать это решение как неуправляемое в первоначальную организацию. В противном случае следуйте инструкциям, содержащимся в диалоговом окне, чтобы отменить экспорт и добавить обязательные компоненты.  
  
5.  На шаге **Экспорт параметров системы (дополнительно)** можно выбрать для включения в решение конкретные параметры системы. Если решение зависит от любых групп параметров системы, выделите их и нажмите кнопку **Далее**.  
  
     См. ниже (**Параметры для экспорта решени1**) дополнительные сведения о параметрах, которые будут включаться в каждом варианте.  
  
6.  На шаге **Тип пакета** необходимо выбрать, следует ли экспортировать решение как **Неуправляемое** или **Управляемое**.  
  
7.  Следующий шаг позволяет выбрать конечное решение для конкретной версии Dynamics 365. Этот параметр обычно используется независимыми поставщиками программного обеспечения, которым может понадобиться экспортировать решение, совместимое с предыдущей версией. Если не планируется импортировать в организацию решение, которое не было обновлено до версии, уже используемой организацией, примите значение по умолчанию.   
  
8.  Выберите **Экспорт**, чтобы загрузить файл решения.  
  
 Точное поведение при загрузке файлов зависит от браузера.
